Up and running in under two minutes.

1
Sign in with Microsoft — use any work or school account. The first sign-in from your tenant triggers a one-time admin consent screen.
2
Choose an entity type — select Users, Groups, Devices, Apps, or Conditional Access from the left rail after sign-in.
3
Search and map — type a name or filter to find objects. Click a result to draw its dependency graph on the canvas.
4
Explore the graph — click any node to see its detail pane. Use the Impact tab on a group to run a pre-delete dependency check.
5
Export findings — download a JSON or CSV impact report from the group detail panel for offline review or ticket input.

EntraMap requests these delegated Graph permissions. All are read-only. Click a permission to see why it is needed.

Reads the signed-in user's basic profile (display name, email, photo). Required to show who is logged in and to anchor the session to a real identity.
Reads the basic profile (name, email, job title) of all users in the tenant. Required to display user nodes on the identity map and support user search.
Reads all groups, their members, owners, and settings. Required to map groups, group membership, nested groups, and group-based license assignments.
Reads all devices registered in the tenant including Intune-enrolled and Azure AD joined devices. Required for the device map and compliance/risk overlays.
Reads all enterprise applications and app registrations including service principals and app role assignments. Required to map application access paths and detect over-privileged roles.
Reads Intune-managed application assignments including target groups. Required to check which groups are targeted by Intune app deployments in the group deletion impact analysis.
Reads all Conditional Access policies including named locations and authentication strength definitions. Required to map CA policy coverage and show which users and groups are in scope.
Reads directory role assignments and role definitions including PIM eligibility and active assignments. Required to show privileged roles held by users, groups, and service principals.
Reads tenant-level organisation details and assigned license SKUs. Required to resolve group-based license assignments to human-readable product names such as Microsoft 365 E3.
Reads access packages, assignment policies, and entitlement catalogs. Required to detect whether a group is referenced in Identity Governance entitlement policies before deletion.
Reads basic information about Microsoft Teams that the signed-in user can access. Required to detect M365 group-backed Teams workloads as blocking dependencies in the deletion impact analysis.
Reads SharePoint site metadata across the tenant. Required to detect groups with associated SharePoint team sites and flag them as active collaboration workloads before deletion.
Reads Planner tasks and plans. Required to detect groups that own Planner plans so that plan ownership can be migrated before the group is deleted.
Reads broad directory objects including administrative units, role templates, and object metadata. Required for the full identity map and to support administrative unit scope detection in the deletion impact analysis.
